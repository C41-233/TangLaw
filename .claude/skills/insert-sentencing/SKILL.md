# 量刑表插入规则

## 黄金法则

**只插入，不动已有。** 绝对不允许修改、移动、删除任何用户已写入的内容和顺序。新条目只能插入到已有条目之间的间隙中，且不改变已有条目的相对顺序。如有疑问，优先将新条目追加到节末。

根据具体罪名的 XML 文档（如 `103-制作御膳失职罪.xml`），将其中的罪刑对应关系插入到 `src/附录/4-量刑表/` 下的适当文件中。

## 操作流程

### 1. 读取罪名文档

读取 XML 文档，重点关注：
- `<law>` 中的条文（简明罪刑）
- 文档正文中的表格（如有总结表）
- `比照适用` 部分的减等规则（如监当官司减一等、东宫减一等、未进御减一等）

### 2. 分析罪刑对应关系

从文档的总结表中提取完整的罪刑矩阵，按以下维度展开：

| 维度 | 取值示例 |
|------|---------|
| 主体 | 主食/工匠、监当官司 |
| 对象 | 御膳/御船、皇太子膳/皇太子船 |
| 罪过 | 误犯食禁/误不牢固、混入污秽之物、拣择不精/未及时整饰、违反食令/缺少设施、不品尝 |
| 进御状态 | 已进御、未进御 |

**"或者"类罪过的拆分：** 如"未及时整饰或者缺少设施"应拆分为"未及时整饰"、"缺少设施"两条独立条目，保留文档中的原始顺序。

### 3. 按刑罚分类汇总（★ 输出前报告）

**在修改文件前**，按刑罚从重到轻列出所有要插入的条目，格式为：

```
绞
{总称}（{主体}{动词}{对象}{罪过}{进御状态}） [{十恶标记}]
流三千里
...
```

按此格式向用户报告完整内容，**等待用户确认后再执行插入**。

### 4. 确定刑罚类型

每个罪刑对应一个刑罚，按以下规则分入对应文件：

| 刑罚 | 文件 |
|------|------|
| 斩、绞 | `1-死刑.xml` |
| 流三千里、流二千五百里、流二千里 | `2-流刑.xml` |
| 徒三年、徒二年半、徒二年、徒一年半、徒一年 | `3-徒刑.xml` |
| 杖一百、杖九十、杖八十、杖七十、杖六十 | `4-杖刑.xml` |

### 5. 定位插入位置

#### 5.1 查找锚点

在目标文件中找到对应刑罚的 `<h2>` 节，在其 `<x-table>` 中找到插入位置。

#### 5.2 插入规则

对于"比照适用"（如御膳→皇太子膳、御船→皇太子船），新条目插入在**对应条目所属的主体块之后**。

例如，如果源文档提到"为皇太子制备御船，比照御减一等"，则：
- 新条目的犯罪主体与对应条目相同（工匠或监当官司）
- 对象从"御船"改为"皇太子船"
- 进御状况（已进御/未进御）不变
- 刑罚减一等

**多犯罪类型共存原则：** 同一刑罚节内有多个犯罪类型（如合和御药、制作御膳、制备御船）时，新类型追加在该节末尾，不与其他类型交叉排列。

#### 5.3 排序规则（多级分块排序）

所有条目按以下规则排序，**同级条目作为一个连续块，不交叉**：

**第一级 — 犯罪主体块：** 工匠/主食 → 监当官司
- 所有工匠/主食条目作为一块，排在前面
- 所有监当官司条目作为一块，排在后面

**第二级 — 对象块：** 御船/御膳 → 皇太子船/皇太子膳
- 同一主体块内，所有御船/御膳条目作为一块，排在前面
- 所有皇太子船/皇太子膳条目作为一块，排在后面

**第三级 — 罪过块：** 误不牢固/误犯食禁 → 混入污秽之物 → 拣择不精/未及时整饰 → 违反食令/缺少设施 → 不品尝

**第四级 — 进御状态：** 已进御 → 未进御

举例：
```
# 正确（分块排列）：
工匠...御船...误不牢固已进御          ← 工匠·御船块
工匠...御船...未及时整饰已进御
工匠...皇太子船...误不牢固已进御      ← 工匠·皇太子船块
监当官司...御船...误不牢固已进御      ← 监当官司·御船块

# 错误（交叉排列）：
工匠...御船...误不牢固未进御
监当官司...御船...误不牢固已进御      ← ✗ 不应在工匠块未完时插入监当官司
工匠...皇太子船...误不牢固已进御
```

### 6. 条目命名格式

```
{总称}（{主体}{动词}{对象}{罪过}{进御状态}）
```

总称从文档标题提取，如`制备御船失职`、`制作御膳失职`；动词如`制备`、`制作`。

示例：
```
制备御船失职（工匠制备御船误不牢固未进御）
制备御船失职（监当官司制备皇太子船未及时整饰已进御）
制作御膳失职（主食制作皇太子膳误犯食禁已进御）
```

### 7. Tab 格式规则

条目使用 Tab 分隔的三列：

```
{罪刑描述}\t{十恶标记/'-'}\t{法条引用}
```

| 列 | 说明 |
|----|------|
| 第1列 | 罪刑描述文本 |
| 第2列 | 十恶标记：`大不敬` / `不道` / `不睦` 等，无十恶则用 `-` |
| 第3列 | 法条引用：`<article>104</article>` |

#### 7.1 Tab 数量计算

Tab 数量取决于文件格式：

**流刑/徒刑文件：** 十恶/`-` 后 1-2 个 Tab，法条引用前
**杖刑文件：** `-` 后 2 个 Tab，法条引用前

Tab 数量需参考同一节中**宽度相近的已有条目**，计算规则：

```
显示宽度 = 中文字符数 × 2 + ASCII字符数 × 1
Tab 移动到下一个 8 的倍数位置
```

**重要：** dash (`-`) 前**必须至少有一个 tab**，即使文本宽度接近 0 列。不可模仿已有条目中 dash 前无 tab 的错误格式。优先匹配同一节中主体相同的已有条目的 Tab 数量。

### 8. 特殊处理

#### 减等规则

- **监当官司**比工匠/主食减一等：在同一对象和进御状态下，监当官司的刑罚比工匠/主食减一等
- **未进御**比已进御减一等：在同一主体和对象下，未进御的刑罚比已进御减一等
- **皇太子船/皇太子膳**比御船/御膳减一等：在同一主体和进御状态下，皇太子对象的刑罚比御对象减一等

#### 十恶标记

- 御膳/御船误犯食禁/误不牢固属于"大不敬"（十恶之六）
- 不品尝、未及时整饰、缺少设施等不属于十恶
- 皇太子膳/皇太子船误犯食禁/误不牢固属于"大不敬"
- 监当官司的犯罪不属于十恶（减等后不构成大不敬）

## 辅助脚本

`insert-template.js` 提供了可复用的 Node.js 工具函数（宽度计算、行查找、条目构建、结构验证等）。

**使用方式：**
1. 将模板复制到项目根目录
2. 在 `insertEntries()` 中填写要插入的条目
3. 运行 `node insert.js`
