# x-table 格式规范

## 基本结构

```xml
<x-table title="表格标题">
列1	列2	列3	...
数据1	数据2	数据3	...
</x-table>
```

- **Tab** 分隔列，可加不定数量 Tab 视觉对齐；解析时连续空 Tab 段被折叠
- `title` 属性 → HTML `<caption>`，无标题则省略
- 不再使用 `type="table"`，未指定 type 时默认按 table 模式处理

## 列合并（`*`）

表头或数据行中 `*` 表示该列合并到前一列（colspan向后合并）：

```xml
<!-- <th colspan="2">罪行</th> + 子罪行为独立单元格 -->
罪行	*	法定刑	十恶	法条
厌魅	意图杀人	皆斩	不道	264
闻丧匿不举哀	*	徒一年	不孝	120
```

- `*` 在表头：表头单元格 colspan（视觉合并）
- `*` 在数据行：该格合并到前一格（等同于 colspan）
- **注意**：`*` 不等同于空格，它代表"合并"

## 空单元格（`-`）

`-` 表示空单元格，渲染为空：

```xml
数据1	-	数据3
```

- `-` **仅代表空值**，不触发 colspan 合并
- `*` 和 `-` 的区别：`*` = 合并，`-` = 空

## 行合并（自动 rowspan）

连续行中**前几列值相同**时，自动合并为 rowspan。合并的条件是：该列**及所有前列**的值在连续行中均相同：

```xml
罪行	*	范围	法定刑	法条
谋反	-	父	绞	248
谋反	-	子年十六以上者	绞	248
```

等效 HTML：
```html
<tr><th rowspan="2">谋反</th><th rowspan="2"/>        <!-- - 触发合并 -->
    <td>父</td><td>绞</td><td>248</td></tr>             <!-- 范围不同导致法定刑法条不合并 -->
<tr><td>子年十六以上者</td><td>绞</td><td>248</td></tr>
```

再如：
```xml
罪行	*	法定刑	十恶	法条
厌魅	意图杀人	皆斩	不道	264
厌魅	意图致人疾苦	皆斩	不道	264
厌魅	为求爱媚	流二千里	不孝	264
```

- 厌魅 3 行中仅有"罪行"列相同（"子罪行"列不同），因此仅 rowspan=3 在"罪行"列
- 子罪行列各自独立
- 法定刑"皆斩"不合并（子罪行列不同）

## 法条列特殊处理

列名为 `法条` 的列，值**用裸数字**，生成器自动转换为 `<article>` 链接：

```xml
法条
248           <!-- 自动转为 <a href="248.html">第二百四十八条</a> -->
120.1         <!-- 自动转为 <a href="120.1.html">第一百二十条之一</a> -->
```

如果法条列已有 `<article>` 标签（如旧数据），应去除，只保留裸数字。

### 非法条列的 `<article>` 必须保留

其他列（如 `备注`、`十恶` 等）中含有 `<article>` 时，**必须保留**，生成器不会自动处理：

```xml
备注
不适用<article>344</article>第一款    <!-- 保留 article 标签 -->
```

## 换行

单元格内换行用 `\n` 表示 HTML `<br/>`：

```xml
祖父母\n曾祖父母\n高祖父母
```

## 转写操作规则（从 HTML table 转换时）

### 表头转换

1. `<caption>` → `title` 属性
2. `<thead>` → 首行 Tab 分隔列名
3. `colspan` → 该列后追加 `*`（每多1列加一个 `*`）

### 数据行转换

4. `<br/>` → `\n`
5. 空 `<td></td>` → `-`
6. **法条列**：去除 `<article>` 标签，仅保留数字
7. **非法条列**：保留原始 `<article>` 标签，不删除
8. 尾随空列可省略不写

### 行合并（不要手动写）

9. **不要**再用 `（...）` 括号语法表示行合并（旧格式已废弃）
10. 直接写数据行，生成器自动按连续同值列合并为 rowspan

### 生成规则（内部实现，了解即可）

- `*` → colspan 合并到前一列
- `-` → 不合并，仅空单元格
- rowspan 条件：该列及所有前列在连续行中值相同
- 法条列裸数字 → 自动 `<article>` 包裹 → 链接化
