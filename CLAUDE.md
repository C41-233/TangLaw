# 项目

本项目是《唐律疏议》的注解网站，通过generator工具将src的内容转化输出到docs，网站入口在docs/index.html。

## 目录结构

- references 参考文献，不应当读取
- src 原始文档
- docs 输出的静态网站页面
- generator 一个dotnet项目，用于将src的原始文本内容转化输出到docs

### src的目录结构

src包含序言、正文、附录、释义。

序言只有 `0.xml`，是序言的全部内容。

正文分为三层，第一级目录是`篇`，第二级目录是`章`，第三级的文档具体的`条。`

目录和文档的命名格式都为 `<number>.<xxx>` 数字表示其序号，xxx表示其标题。`0.xml`是特殊的文档，表示该级目录本身的内容。

### 文档结构

`<src>` 内是古文原文。

`<law>` 内是条文。

`>` 标记了一级标题，`>>` 标记二级标题，以此类推。

### docs的目录结构

styles是样式目录，contents是generator的输出目录，里面包含所有的输出html文档。

generator负责在contents中生成index.html以及其他详情页面。

index.html是网站入口，从上到下依次包含：

- 标题 `《唐律疏议》的理解与适用`
- `正文`的目录（序言包含其中），按照篇、章、条的层级，点击可以跳转到详情页面。
- 律文，合并显示所有正文的律条（law），也包括序言。

详情页面的内容，从上到下依次是：

- 标题，即本条的标题。
- 条文，对应`<law>`中的内容。
- 正文，xml文档的主要部分内容，即law和src以外的所有内容。
- 原文，对应`<src>`中的内容。

详情页面的标题，根据其所处的层级和序号，要显示为 `第一篇 名例律` `第一章 刑罚` `第一条 笞刑`，注意中文数字，对目录同样如此。序言只显示为`序言`。

## 基础规则

- 不允许执行任何 `git `命令。
- 不允许删除任何文件。
